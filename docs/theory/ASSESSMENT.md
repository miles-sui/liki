# Fivefold Types — 评估逻辑

> **评估是采集。算法是加工。题目是原料——多少都能处理。**

本文档描述 Fivefold Types 的评估算法。理论见 [THEORY](THEORY.md)，编码规范见 [SPEC](SPEC.md)。

## 核心原则

### 评估与算法解耦

题目是数据采集层。算法是数据处理层。两者完全独立：

- **题目可以换、可以增减、可以定制。**
- **算法不关心题目具体是什么。** 只要作答按规范格式上报，算法统一处理。

### 多评者、多题目——来者不拒

同一个人可以被任意数量的评者评价。每一条作答是独立的数据点。全部汇总 → 统一计算。

## 形状评估（d）

### 输入格式

三元自由选：每题 3 个选项（3 个不同元素），选符合的任意个（0–3 个）。

```json
{
  "qid": "Q01",
  "selections": ["Wood", "Fire"]
}
```

### 聚合计分

```
raw[e] = Σ 所有评者所有题的 (e 被选中 ? 1 : 0)

p[e] = raw[e] / Σraw                ← 展示比例 (Σ=1)
d[e] = 5 × (p[e] - 0.2)             ← 形状 (Σd = 0)
```

- 每次选中各计 1 分。不区分排名。
- 每个评者每条作答独立累加，不分组、不加权。
- 纯 raw 计分，零参数。无平滑、无先验、无基底修正。
- Σd = 0 由 Σp = 1 自动保证。
- ×5 缩放：p 偏移 ±0.2 → d 偏移 ±1.0。d[e] = +1.0 表示该元素拿了 40% 的加权分。

等权 (1:1)：每次选中权重等同，不追问"哪个更像我"。

### p=0% 的含义

p[e] = 0% 可能出现——该元素在所有作答中从未被选。这不表示"你没有该元素"，而精确表示"在追选格式下，该元素从未在任何题中被选中"。

- 三选项格式每次只出现 3 个元素。排序靠后的元素在稀疏数据中可能永远轮不到——这是测量格式的固有局限，不是人的缺陷。
- 五行描述的是行为偏好的**相对偏移**——在自由选格式下，你倾向于认同哪些元素。0% 只说明该元素在你的认同清单中排在末尾。
- 不做数值修正（不加基底、不平滑）——测到什么就是什么。诚实比光鲜更有价值。

### d / p 同源

d 和 p 同源于 raw。两者满足 p = 0.2 + d/5——同源的代数恒等式，非互推。内部运算始终用 d，对外展示用 p。

### 平局

最高元素并列 → d 如实反映。报告描述"木火并重"。

### 不均匀评者自动平衡

评者多的题自然贡献大——多人评的题信度更高。**人越多 d 越准。**

## 评者流程

### 自评与他评分桶

```
自评作答 → raw_self → d_self
他评作答 → raw_other → d_other
```

输出：
**自评剖面** — 始终展示
**他评聚合剖面** — 1+ 人时展示
**差异视图** — Δd = d_self - d_other

Δd[e] 显著为正 → 自我认知高于他人观察。Δd[e] 显著为负 → 他人看到的强于自评。

### 信度标注

| 评者数 | 标注 |
|--------|------|
| 0 | 不展示他评层 |
| 1-2 | "剖面仅供参考" |
| 3-5 | "剖面初步收敛" |
| 6+ | 无前缀 |

## 选题逻辑

题库共 30 道，按 6 轮组织，每轮 5 题为一个平衡单元。每轮推送，提交后可继续下一轮。同一算法用于自评派题和他评智能派题。选题按三级优先级：

1. **补缺**：推送用户未答过的轮次。做满 6 轮后不可重复评估
2. **覆盖**：在未答轮次中，优先总体回答数低的轮次
3. **争议**：在未答轮次中，优先已有回答中分歧大的轮次

选题算法以轮为单位推送，保证每轮内的元素出场平衡。

选题算法的实现细节见 [FUNC-SPEC](../FUNC-SPEC.md)。

## 测量边界

### 自由度

d 向量 Σd=0，5 个坐标中只有 4 个独立——任意一个可由其余四个线性推导。这不影响个体内分析和 Bond 运算（这些运算保持 Σ=0），但影响因子分析验证的方法选择。常规 5 因子 CFA 在 ipsative 数据上会因协方差矩阵奇异而失败——需使用 ILR 变换或专门处理组合数据的模型。

4 个自由度是 d 向量 Σd=0 约束的数学后果，不是五行理论对五元素独立性的否定。理论假设五个独立维度，测量捕捉的是它们在有限选项下的相对认同优先级。两者不在同一数学空间——这是测量层对理论层的近似，不是等同。

### 信度指标

| 指标 | 方法 | 说明 |
|------|------|------|
| d 余弦相似度 | 间隔 2-4 周，d 向量余弦相似度 | 基础信度指标——反映剖面形状整体稳定性 |
| 25 型分类一致率 | 间隔 2-4 周，同一被试分类结果一致的比例 | 用户面指标——反映用户直接感知到的可靠性 |

除 r 值外，同时检查分类一致率。两者同源——高余弦一致率意味着高分类一致率，但分类在跨 Voronoi 边界时可能出现不一致（即使余弦相似度高）。

### 验证计划

- **零信号基线**：首批数据收集后运行 Monte Carlo 模拟，确定纯随机响应下的 d 抽样分布，作为信号检测的基线
- **Split-half**：前 3 轮与后 3 轮分别计算 d，验证剖面形状不受轮次顺序影响

### 自评-他评收敛

不设固定阈值。随评者数增加，自评-他评 d 相关系数应单调递增并趋近重测信度的上界：

| 评者数 | 预期 |
|--------|------|
| 1-2 | 低收敛——"剖面仅供参考" |
| 3-5 | 初步收敛——"剖面初步收敛" |
| 6+ | 充分收敛——应接近重测信度 |

### 表面效度

| 指标 | 方法 | 目标 |
|------|------|------|
| 表面效度 | "这段描述像我" 认同率 | ≥ 75% |
