# Fivefold Types — 五行维象 — 核心理论

> **五行为体。生克为用。力有流向。**

本文档定义 Fivefold Types 的核心理论——基于中国五行生克代数的连续维度人格-关系模型。编码规范见 [SPEC](SPEC.md)，评估设计见 [ASSESSMENT](ASSESSMENT.md)，文化定位与跨文化考量见 [CULTURE](CULTURE.md)，创新声明见 [NOVELTY](NOVELTY.md)。

## 模型总览

### 一个人的定义

一个人由一个形状向量刻画：

```
d = [d_W, d_F, d_E, d_M, d_R],   Σd = 0

d_i > 0  →  偏盛 (elevated)
d_i < 0  →  偏弱 (subdued)
d_i ≈ 0  →  中平 (balanced)
```

d 回答"在你内部，木火土金水的相对强弱"。零是中性点——不是理想状态，只是形状的坐标原点。Σd=0 是形状的数学定义：整体水平分离后，剩下的是纯相对结构。

内部运算用 d（Σd=0，生克守恒），对外展示用 p（Σp=1）。d 和 p 同源于 raw——纯统计计分，零参数。两者满足 p = 0.2 + d/5，这是同源的代数恒等式，非互推。详见 [ASSESSMENT](ASSESSMENT.md)。

**测量格式**：d 由 30 题三元追选（3 选 2 = 排除 1）采集。6 轮 × 5 题，轮内元素出场平衡，全局每对元素恰好 9 次同题比较。d 有 4 个独立自由度（Σd=0 消耗 1 个）。五行理论假设五个独立维度，测量捕捉的是它们在追选约束下的相对调用优先级——这是测量层对理论层的近似。详见 [QUESTIONNAIRE](QUESTIONNAIRE.md) 和 [ASSESSMENT](ASSESSMENT.md)。

### 两个矩阵承载全部运算

```
S (生):  木→火→土→金→水→木    5×5 置换矩阵
C (克):  木→土→水→火→金→木    5×5 置换矩阵
```

### 一气周流

中医五行气运模型中，五行非静止属性，而是同一个元气在五个阶段的流转形态：

```
          火（上散）
         /        \
      木（左升）  金（右降）
        \        /
      水（下藏）← 土居中斡旋
```

- 木火为**发**（往外放）：木左升启动，火上散推至高峰
- 金水为**收**（往内拉）：金右降收敛，水下藏沉淀蓄能
- 土居**中**斡旋：方向转折的轴，让发收完成切换，不往任一方向偏

五化（生长化收藏）即一气在五个阶段的形态名。

## 元素人格定义

本节将经典共识翻译为可直接指导文案写作的人格基准。五个角色各含：核心冲动、日常行为画面、健康态与过载态、复合型中的主/次语感。

### 木 W — 开创者 Pioneer

**核心冲动**：向前推进。不动就难受，动起来反而踏实。

**行为画面**：别人还在想，你已经开始做了。不是你不思考——是你的思考发生在行动中。坐久了焦躁，一动手脑子反而清楚。

**健康态**：率先破局、带动他人。事情因你的存在而从零变成一。
**过载态**：停不下来。该等的时候也推。伤在没耐心——对别人，也对自己。

**作主时**：你的底色是启动和推进。
**作次时**：（生序）给你行动力的底盘；（克序）帮你把冲动变成有耐心的推进。

---

### 火 F — 发光者 Luminary

**核心冲动**：在场辐射。你的光不挑人——你在哪里，哪里就被照亮。

**行为画面**：你走进来，场子温度就不一样。不是因为你说了什么、做了什么——是你本身在，空气就变了。你的光不挑人：所有人都在你的光亮里，不是因为你选中了他们，是因为你的光本来就是满的。

**健康态**：被你在场感染的人感到被真正看见，不是被评价、不是被消费。
**过载态**：光一直亮着，忘了自己也需要被人看见。伤在一直往外辐射，没人给你回。

**作主时**：你的底色是在场辐射，自然照亮。
**作次时**：（生序）给任何元素注入温度和感染力——让它被看见；（克序）给任何元素添上在场感——让它不止于冷判断。

---

### 土 E — 培育者 Cultivator

**核心冲动**：接住、养好、交出去。把原料变成收成。

**行为画面**：你不是场上最显眼的。但你是每天到、每天做、不跳步骤的那个。别人把烂摊子丢给你，过一阵子，烂摊子变成了能用的东西。不是魔法——是日复一日不放弃。

**健康态**：你培育人和事的能力让周围感到踏实。别人信任的不是你能兜底，是你能把东西养好、送到下一个阶段。
**过载态**：太能接、太不拒绝。谁都找你这个"靠谱的人"，最后你手里一堆别人的烂摊，没人帮你收拾。伤在不会说"不"。

**作主时**：你的底色是接住——转化——输出。居中运作。
**作次时**：（生序）给任何元素接地，让它不飘；（克序）给任何元素设一条河岸，让它不泛滥。

---

### 金 M — 规范者 Refiner

**核心冲动**：过筛。东西好不好、话对不对——你一过就知道。筛子本身不够好，你也敢拆了重铸。

**行为画面**：你天生有根准绳。不一定说出来，但不过关的东西在你这过不去。别人开始可能觉得你挑剔，后来发现你点过头的事基本不会出错。你那根准绳不是死的——标准不对，你连标准一起翻新。你不是只守规矩的人，你是立规矩的人。

**健康态**：你手里的标准让周围信任。你不说废话、不做废事，身边人也因为你的存在而提升了输出质量。
**过载态**：标准变成苛责——对自己，也对别人。什么都能挑出刺来，包括自己做的事。伤在永远觉得不够好。

**作主时**：你的底色是判断、筛选、革新。
**作次时**：（生序）给任何元素装上骨架——让它从散漫变成有结构；（克序）给任何元素装上瞄准镜——让它不乱动；也让它知道什么时候该换一把尺子。

---

### 水 R — 蕴蓄者 Reservoir

**核心冲动**：往下沉。让一切自然渗透进来。积蓄，然后贯通。

**行为画面**：你不是不合群——你需要时间沉下去。独处对你不是孤独，是回血。别人浮在表面，你整个人泡在深处——不是一直在想，是让东西自己渗透、自己连起来。出来的，是别人没看到的那个点。话少，但一说，场上就安静了。

**健康态**：你是周围人的深度来源。别人在表面忙，你在底下连好了。
**过载态**：沉得太深，出不来。独自待太久，和外界断线。伤在别人不知道你在泡什么，你自己也忘了怎么浮上来说话。

**作主时**：你的底色是渗透——浸泡——积蓄——贯通。
**作次时**：（生序）给任何元素注入深度储备——让它出手前经过了深水层的浸泡；（克序）给任何元素淬火——让它不过热、不过锐。

## 公理

以下陈述是模型的输入假设，非待验证命题。模型的预测（Bond 力流方向、Flow 季节效应）可被检验，这些陈述本身是定义：

**S 和 C 是 5×5 置换矩阵**，承载全部生克运算。S[j][k]=1 ⇔ 元素 j 生元素 k；C[j][k]=1 ⇔ 元素 j 克元素 k。完整定义见 [SPEC](SPEC.md)。

**形状空间是 Σ=0 的超平面**。d ∈ ℝ⁵，Σd_i = 0。所有运算（生克、Bond、Flow）保持 Σ=0 不变。d 描述元素间的相对强弱，不含绝对水平。

**只有偏盛的元素对外输出力**。d⁺ = ReLU(d) = max(d, 0)。偏弱元素（d_i < 0）受力但不主动发力。这是"偏盛者发力"的数学表达。

**25 个原型在单纯形棱上（d-空间，Σd=0）**。5 个纯型在顶点，20 个复合型在棱上，由统一参数 α=0.52 定位（p_proto = [α, 1-α, 0, 0, 0]）。α 由 Monte Carlo 优化确定，使 25 型分布接近均匀。详见 [SPEC](SPEC.md)。

**solar month table 为 12 个 one-hot 向量**，每月对应一个旺相元素，与 S、C 同质——只指明方向，不量化。详见 [ENGINE](../ENGINE.md)。

**零参数**。以上所有量均为硬编码常量。没有可调系数、权重、阈值。模型行为完全由 d 的实测值决定。

## 内在动力

### 生克公式

只有偏盛的元素主动输出生克力。偏弱的元素自顾不暇，不主动生人也不主动克人：

```
d⁺_i = max(d_i, 0)                                 只有偏盛者发力

d_eff = d + Sᵀd⁺ - Cᵀd⁺

守恒: Σd_eff = Σd = 0
```

Sᵀd⁺ - Cᵀd⁺ 并非 d 以外的外加修正——d 本身就蕴含了生克关系。木 0.6 不是说"木本身 0.6 然后额外生火"——木的 0.6 本身就包含了对火的推动力。公式只是把这个隐含力显式化。

- **生（facilitation）**：生我者对我的推动。木偏盛 → 火被推动。行为倾向间的协同。
- **克（restraint）**：克我者对我的约束。水偏盛 → 火被约束。行为倾向间的权衡。

只有自身有余力，才能对外输出影响——偏盛才有余力。偏弱的元素在生克中是被动的：它们被生、被克，但不主动输出。

#### 为什么 ReLU

去掉 ReLU（d⁺ = d 即全分量输出）时，偏弱元素也在主动生人克人——偏弱元素（如 d_i = -0.3）对外输出的生克力为负，等价于反向生克（克变成生，生变成克），这与"偏盛者发力"原则矛盾。

ReLU 之外的可选方案——softplus、sigmoid、带温度的阈值——都会引入至少一个可调参数。softplus 需要一个平滑参数 β，sigmoid 需要一个中心偏移和一个温度参数。这些参数要么从数据中估计（引入统计自由度损失），要么手工设定（引入主观性）。ReLU 是唯一同时满足以下条件的截断函数：(a) 零参数，(b) 在零处有一个清晰的"发力/不发力"边界，(c) 线性保留正值的幅度。条件 (b) 是模型的核心——有余力就是有余力，没有就是没有。不存在"有一点点余力所以输出一点点力"的中间状态——零处硬边界是"偏盛者发力"原则的精确数学表达。

### 分类

Profile 和 prototype 统一在单纯形 d-空间（Σd=0）。分类使用余弦相似度——方向匹配，不受幅度影响：

```
identity = argmax_{k}  cos(d, d_protoₖ)
         = argmax_{k}  d · d_protoₖ / (‖d‖ · ‖d_protoₖ‖)

pure (×5):   W, F, E, M, R — 顶点
composite (×20): WF/FW, FE/EF, EM/ME, MR/RM, RW/WR,
                 WE/EW, FM/MF, ER/RE, MW/WM, RF/FR — 棱上，首字母为主
```

不需要 ReLU。余弦相似度自然分离了"形状方向"和"形状强度"——纯型的 d 模长天然大于复合型，点积会偏袒纯型；余弦消除了这种幅度偏袒。

平局时选主导元素（首字母）在 d 中取值最大者。平局仅发生在 Voronoi 边界（测度零集），对分布无统计影响。

### 为什么是 α=0.52

纯型在 5-单纯形的顶点上，每个顶点连接 4 条棱。复合型在棱上，每条棱有 2 个方向。分类器的 Voronoi 划分取决于原型在棱上的位置参数 α：

- α → 0.5：原型趋近棱中点，纯型 Voronoi 区域被压缩
- α → 1.0：原型趋近顶点，纯型和同方向复合型趋同

α 由 Monte Carlo 在 triad 追选模拟下确定。选定 α=0.52：纯型 ~19%（约 1/5），复合型 25 类在 5 个几何组内均匀分布，元素平衡完美（W/F/E/M/R 各 ~20.0%）。完整数据见 [SPEC](SPEC.md)。

## Bond — 关系动力

### 核心公式

B 的 d⁺ 直接通过 S 和 C 矩阵作用在 A 上：

```
Δ^A = Sᵀd^B⁺ - Cᵀd^B⁺
d^A_eff = d^A + Δ^A
```

与内在生克同一套公式。B 的偏盛元素就是作用于 A 的力源。ΣΔ^A = 0，d^A_eff 仍在 Σ=0 空间内。

Δ_i > 0 → B 在推高 A 的该元素。Δ_i < 0 → B 在拉低 A 的该元素。不判好坏——用户自己判断这段力对自己是利是弊。

A 对 B 对称计算，下标对调。

## Flow — 时间维度

### 季节调制

solar month table 为 12 个节气月到主导元素的 one-hot 映射（t ∈ {0,1}⁵, Σt=1）。与 S、C 矩阵同质——只指明方向，不量化。

季节效应为纯方向指示，不叠加数值偏移：

```
generates  = S · t_month    → 旺相元素所生之元素 ↑
restrains = C · t_month    → 旺相元素所克之元素 ↓
```

用户的 d 剖面不随季节变化。每个节气月只在被生元素上标记 ↑（生力方向），被克元素上标记 ↓（克力方向）。季节效应的强度因人因时而异，无法统一定量——本模型只呈现方向，不猜测强度。

各月旺相元素及生克方向：

| 月份 | 旺相 | 生 ↑ | 克 ↓ |
|------|------|------|------|
| 寅卯 (春) | 木 | 火 | 土 |
| 巳午 (夏) | 火 | 土 | 金 |
| 辰未戌丑 (四季) | 土 | 金 | 水 |
| 申酉 (秋) | 金 | 水 | 木 |
| 亥子 (冬) | 水 | 木 | 火 |

节气月源于东亚季风气候区的农耕历法。辰未戌丑为四季土月——每季末月归土，源于传统五行旺相休囚死理论。南半球与赤道的处理策略、跨文化考量见 [CULTURE](CULTURE.md) §5.1。

### 年历

同一份 d 贯穿全年，不逐月变化。各月仅生克方向轮转。不是预测——是指示。

## Concord — 人际生克（顺逆平）

### 核心公式

两个人 A 和 B 之间的生克力同样只有偏盛元素参与。使用 ReLU(d) 截取偏盛分量后，通过对称生克矩阵计算双向合力：

```
sh = d_A⁺ · (S + Sᵀ) · d_B⁺     生力 — 双向推动
ke = d_A⁺ · (C + Cᵀ) · d_B⁺     克力 — 双向约束
```

S+Sᵀ 和 C+Cᵀ 是对称化矩阵——生克关系从单向置换变为双向通道。只取偏盛分量（d⁺ = ReLU(d)）意味着只有双方偏盛的元素才参与关系运算。

### 判定

```
若 sh > ke + ε  →  顺 (concord)   — 生力主导，相互推动
若 ke > sh + ε  →  逆 (counter)    — 克力主导，相互制约
否则           →  平 (neutral)    — 生克均衡
```

ε = 0.02 为判定阈值，防止微小数值波动导致分类震荡。同型自配对大部分落在平区（~85%），符合"与自己相处无额外张力"的直觉。

### 设计理由

**为什么是 ReLU(d) 而非 full d**：与内在生克一致——只有偏盛的元素有余力对外输出。偏弱元素的生克力为零，不参与关系运算。这避免了"偏弱元素反向生克"的语义矛盾。

**为什么是 S+Sᵀ 和 C+Cᵀ**：关系是双向的。A 生 B 和 B 生 A 应同时计入生力——顺是"两人互相成就"的和。克亦然。

**ε 的角色**：零参数纪律要求 ε 不可调。0.02 是统计校准结果——在 25 型随机配对下给出约 40/40/20（顺/逆/平）的总体分布。详见 [SPEC](SPEC.md)。

---

## 设计原则

**五行是根。** 回归生克代数本身。
**偏盛者主。** 只有偏盛的元素主动输出生克力——ReLU 切掉噪声。
**描述不评判。** 力流分析不判好坏，用户自己判断。
**连续内核，离散传播。** 计算保留完整精度，展示收敛为可理解的标签。
**独立于出生日期。** d 来自行为观察，不是宿命。
**零参数。** S 和 C 之外没有可调旋钮。没有系数，没有权重。
