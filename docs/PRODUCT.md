# 产品规格

25types 的产品定位、核心产品、SEO 获客矩阵与变现模型。本文档是所有工程决策的"为什么"。

## 1. 产品定位

**25types = 人生答案引擎。** 用户不是在找"测试"——用户在找答案。

**核心产品：报告。** 用户输入出生信息，获得命盘数据（免费），然后可以问任何问题——感情、事业、起名、择日。引擎计算 ground truth，LLM 用自然语言解释给用户。

> "我和他能结婚吗？"
> "孩子叫李什么好？"
> "5 月跳槽好不好？"
> "我适合学医还是学计算机？"

同样的出生信息，到哪算都是同一个结果。**AI 能聊，但只有我们能算。** ChatGPT 算不对八字——它会幻觉。我们的引擎给出可验证的正确数据，LLM 只负责表达。

### 竞争差异化

| 用户感知 | 竞品 | ChatGPT | 我们 |
|---------|------|---------|------|
| 命盘数据准确吗？ | 部分隐藏 | **经常算错** | 可验证——拿去别处验 |
| 能自由提问吗？ | 固定报告 | 能，但数据是假的 | 能，且数据正确 |
| 答案能溯源吗？ | 不透明 | 不能 | 每句解读可追溯到引擎数据 |
| 不满意能退吗？ | 否 | 否 | 24 小时退款 |

我们是唯一"可验证正确数据 × 自由提问"的产品。

## 2. 核心功能：报告（/app#ask）

### 2.2 报告结构

每份报告 = 引擎数据（结构化表格/列表，不可变）+ LLM 文案（自然语言解读，锚定在引擎数据上）+ 溯源块（强制）。

| 报告类型 | 引擎提供 | LLM 提供 |
|---------|---------|---------|
| 命格解读 | 排盘、五行统计、大运表、用神格局 | 命格判断、五行分析、大运叙事、发展建议 |
| 起名 | 五行缺补、候选字列表、三才五格评分 | 字源典故、寓意音韵、名字详解 |
| 择日 | 候选日期排序、冲合关系、分值 | 逐日解释、建议时段、综合推荐 |
| 关系 | 合盘7维分数、Bond Delta、Concord | 关系叙事、摩擦点、相处建议 |
| 职业 | 学科/职业映射表 | 方向解读、发展建议 |

### 2.3 报告模型

**一问一报告，无对话。** 用户提一个问题，系统生成一份完整报告。每次 LLM 调用只包含当前问题——没有对话历史，没有上下文管理。引擎数据完整注入 system prompt。

**不满意 = 修改重问。** 用户点"修改重问"→ 输入框预填旧问题 → 用户修改 → PUT 更新同一份报告。新请求中附带上一版摘要 + 用户反馈，LLM 知道"用户在上一版基础上想要什么调整"。

**新问题 = 新报告。** 用户想换个话题 → 点"新问题"→ 输入新问题 → 生成新报告。历史报告保留在侧栏列表中，可随时回看。

## 6. 信任与转化机制

### 6.1 数据全貌先看

排盘数据、五行分布、大运表——完整免费展示，不隐藏。用户可截屏去小红书验算。信任数据了，自然信任基于数据的解读。

### 6.2 答案预览：完整样品

预览不是截断——是一个完整 insight 的展示。用户读完后付费，是因为"这个质量我想要更多"。

### 6.3 24 小时退款

每个付费答案附"不满意？24 小时内联系退款"。竞品没有一个敢退——先做到的那个就是"诚实的那个"。

### 6.4 数据溯源

每句 LLM 解读标注引擎数据来源。"这条解读基于：日主甲木 + 寅月身强 + 用神金。" 用户不是盲信 AI——是 AI 在帮你读你自己的数据。

### 6.5 转化心理路径

1. 免费数据 → "这算法算得对"
2. 预览一个完整答案 → "这质量不错"
3. 退款承诺 → "不满意可以退"
4. 价格 ¥9.9 → "冲动消费，不纠结"

每一步消除一个顾虑。

## 7. 增长引擎

### 7.1 Bond 社交裂变

Bond 天然是社交对象——"我和 TA 的关系"比"我的性格类型"更有分享动机。分享卡片：两人类型 + 生克方向 + 一句解读。扫码 → 对方做评估 → 自动计算两人 Bond → 进入报告。

### 7.2 每日建议 + 每日提问 + 月流推送（留存）

**每日建议（默认）：** 系统根据今日干支 + 用户命盘，每天自动推送一条轻量建议。建议类型轮换：穿搭颜色 → 行事方向 → 情绪提示 → 健康小贴士。≤150 字，免费。用户打开 App 即看到，无需提问。

**每日提问（从建议触发）：** 用户看了建议后，可以点"问今天的事"提一个和今天相关的问题。每日限 3 次免费。严格限定"今天"范围——超出引导到完整报告。建议是钩子，提问是互动，报告是转化。

**月流推送：** 每月初一推送："本月辰月，土旺。你的日主甲木——土是你的财星，适合做财务决策。"免费，简短。用户有需求时自然回来。

落地页：`/en/assess`（personality test）、`/en/career`（career aptitude test）。

**关系分析（25types 引擎侧）— 普世，8 个用例：**
恋爱匹配、婚前解读、分手复合、婚姻修复、亲子关系、朋友匹配、子女天赋、合伙人匹配。
落地页：`/en/compatibility`（relationship compatibility）。

**八字轻量（命理引擎）— 窄众但存在，4 个用例：**
八字排盘（"Chinese astrology chart"）、合婚（"Chinese zodiac compatibility"）、结婚择日（"auspicious wedding date"）、养生（"TCM body constitution"）。
落地页：`/en/mingli/bazi`（工具壳——只做排盘，不放深度解读 SEO 内容）。

**定向择路（25types 引擎侧）— 有需求，2 个用例：**
定居城市（"where should I live"）、留学目的地（"where should I study abroad"）。

### 没有需求的用例——不做

| 产品线 | 为什么没有 |
|--------|-----------|
| 起名命名（整条线） | 英美父母按家族传统/读音/流行度起名。五行起名在英文世界不存在。PRODUCT.md §2 已明确——"不做西方父母五行起英文名" |
| 择时择日（除结婚外） | 开业/签约/搬家/手术/考试/离职——日期由律师、医生、HR、学校决定，没有择日文化 |
| 八字深度解读 | 命格/大运/流年/财运/事业运/正缘——八字概念（十神、用神、大运）没有英文认知土壤 |
| 中式关系用例 | 婆媳/学区/升学方向/择校——是中国社会结构的产物，英文世界不存在对应需求 |

| `career` | — | ✓ | ✓ |
